SCREEN TIME TRACKER — HOW IT WORKS
====================================

WHAT IS IT?
-----------
A local macOS app that tracks how much time you spend on LeetCode, LinkedIn,
and Gmail in Google Chrome. It shows your stats on a live dashboard in your
browser at http://localhost:8050.


PURPOSE
-------
To help you understand and visualize where your time goes on Chrome — specifically
on job prep (LeetCode), networking (LinkedIn), and email (Gmail). It gives you
daily, weekly, and all-time breakdowns so you can stay accountable.


TECH STACK
----------
- Python 3           → main programming language
- Flask              → lightweight web framework that serves the dashboard
- SQLite             → file-based database (screentime.db), no server needed
- AppleScript        → macOS scripting to read the active Chrome tab
- HTML / CSS / JS    → frontend dashboard (vanilla, no frameworks)
- launchd            → macOS built-in service manager for auto-start


HOW IT WORKS INTERNALLY
-----------------------

1. TRACKER (tracker.py)
   - Runs a loop every 5 seconds
   - Calls AppleScript via the "osascript" command to ask macOS:
     "What app is in the foreground? If it's Chrome, what's the tab title and URL?"
   - Checks if the tab URL or title contains keywords:
       "leetcode"      → logs as LeetCode
       "linkedin"      → logs as LinkedIn
       "gmail"         → logs as Gmail
       "mail.google"   → logs as Gmail
   - If matched, writes a row to the SQLite database:
       timestamp, date, category, tab title, duration (5 seconds)
   - If Chrome isn't active or the tab doesn't match, it does nothing

2. DASHBOARD (app.py)
   - Flask web server running on localhost port 8050
   - Has two API endpoints:
       /api/summary   → returns total seconds per category for today/week/all
       /api/daily     → returns daily totals for the past 7 days
   - The frontend is a single HTML page with embedded CSS and JavaScript
   - JavaScript calls these APIs every 10 seconds and redraws the charts
   - Shows: time cards per site, bar chart, and a 7-day timeline

3. AUTO-START (launchd plists)
   - Three .plist files registered in ~/Library/LaunchAgents/
   - macOS reads these on login and starts the processes automatically
   - com.screentime.tracker     → runs tracker.py in background
   - com.screentime.dashboard   → runs app.py in background
   - com.screentime.morning-open → watches for Chrome to launch, then opens
                                    the dashboard in your browser (once per day)


DATA FLOW
---------

  Chrome tab
      ↓ (AppleScript reads title + URL)
  tracker.py
      ↓ (classifies and writes)
  screentime.db (SQLite file)
      ↓ (Flask reads and queries)
  app.py (API endpoints)
      ↓ (JSON responses)
  Browser dashboard (auto-refreshes every 10s)


IS IT SAFE?
-----------
- 100% LOCAL — nothing leaves your Mac. No internet, no cloud, no analytics.
- No passwords or sensitive data is stored — only tab titles, URLs, and timestamps.
- The database is a single file (screentime.db) in the project folder.
- AppleScript only reads the active tab info — it cannot modify anything.
- The dashboard runs on localhost only — no one else on your network can access it.
- All code is open source in the repo — you can read every line.


DOES IT RUN ALL THE TIME?
-------------------------
- It starts automatically when you log into your Mac (via launchd).
- It stops when you shut down or sleep your Mac.
- When your Mac wakes up, it starts again automatically.
- When Chrome isn't active or you're on a non-tracked site, the tracker
  just checks and skips — it uses almost zero CPU and memory.
- You can fully stop it anytime by running: bash uninstall.sh
- You can restart it anytime by running: bash setup.sh


FILES
